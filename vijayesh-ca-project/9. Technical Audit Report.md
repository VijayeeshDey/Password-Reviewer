1.2 WHY MD5 IS A "CRITICAL RISK"
─────────────────────────────────────────────────────────────────────────────

MD5 (Message Digest 5) was designed in 1991 as a hash function, but has been
considered BROKEN since 2004 when researchers demonstrated practical collision
attacks.

WHY MD5 FAILS FOR PASSWORD STORAGE:

┌─────────────────────────────────────────────────────────────────────────┐
│                         MD5 VULNERABILITIES                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1. SPEED IS THE PROBLEM                                               │
│     └─ MD5 can compute ~10 BILLION hashes per second on a GPU          │
│     └─ Modern GPUs have 10,000+ cores processing in parallel            │
│                                                                         │
│  2. NO BUILT-IN SALT                                                   │
│     └─ Same password ALWAYS produces same hash                         │
│     └─ Rainbow table attacks work instantly                            │
│     └─ Pre-computed lookup tables reverse hashes in milliseconds        │
│                                                                         │
│  3. KNOWN COLLISIONS                                                  │
│     └─ Two different inputs can produce same hash                       │
│     └─ Security proof invalidated                                      │
│                                                                         │
│  4. NO WORK FACTOR                                                    │
│     └─ No computational cost to slow down attackers                     │
│     └─ Single hash computation takes microseconds                      │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘

REAL-WORLD IMPACT:
─────────────────────────────────────────────────────────────────────────────
• RockYou 2009 breach: 32 million MD5 hashes → 60% cracked instantly
• LinkedIn 2012 breach: 6.5 million SHA-1 hashes → 60% cracked within days
• Ashley Madison 2015: bcrypt used → still mostly uncracked

Our simulation shows: admin and jsmith hashes can be cracked in < 0.001 seconds

1.4 SHA-256: BETTER BUT NOT FOR PASSWORDS
─────────────────────────────────────────────────────────────────────────────

SHA-256 is cryptographically stronger than MD5/SHA-1 but STILL UNSUITABLE
for password storage because:

• NO SALT - same password = same hash
• TOO FAST - millions of hashes/second
• NO WORK FACTOR - no computational cost to slow attackers
• GPU-OPTIMIZED - easily parallelized on graphics cards

mrodriguez's SHA-256 hash is BETTER than MD5/SHA-1 but still at risk.


1.5 WHY BCRYPT IS SECURE
─────────────────────────────────────────────────────────────────────────────

bcrypt (1999) was designed specifically for password hashing and remains

2.2 HASHCAT COMMANDS FOR CRACKING
─────────────────────────────────────────────────────────────────────────────

Hashcat is the world's fastest password recovery tool, optimized for GPU.

```bash
# ========================================
# CRACKING MD5 HASHES
# ========================================

# Method 1: Dictionary Attack (Fastest)
# - Uses a wordlist like rockyou.txt
# - Best for common passwords
hashcat -m 0 -a 0 hashes.txt rockyou.txt

# Explanation:
# -m 0    = MD5 hash mode
# -a 0    = Dictionary attack mode
# rockyou.txt = Common password wordlist

# Method 2: Dictionary + Rules (Better coverage)
# - Applies mutations to wordlist words
# - Tests: password → P@ssw0rd, password123, etc.
hashcat -m 0 -a 0 hashes.txt rockyou.txt -r rules/best64.rule

# Method 3: Brute Force (Slow, exhaustive)
# - Tries ALL combinations
# - Only practical for short passwords (1-6 chars)
hashcat -m 0 -a 3 hashes.txt ?a?a?a?a?a?a

# ?a = all printable characters
# ?d = digits only (0-9)
# ?l = lowercase only
# ?u = uppercase only
# ?s = special characters


# ========================================
# CRACKING SHA-1 HASHES
# ========================================

# Dictionary Attack
hashcat -m 100 -a 0 hashes.txt rockyou.txt

# -m 100  = SHA-1 hash mode
# Other flags same as MD5


# ========================================
# CRACKING SHA-256 HASHES
# ========================================

# Dictionary Attack
hashcat -m 1400 -a 0 hashes.txt rockyou.txt

# -m 1400 = SHA-256 hash mode
# Note: Still fast due to no work factor


# ========================================
# CRACKING BCRYPT (ALMOST IMPOSSIBLE)
# ========================================

# With cost factor 10, this would take YEARS
hashcat -m 3200 -a 0 hashes.txt rockyou.txt

# Estimated time: 3-10+ years per password
# Even with GPU cluster, not practical
```

```python
# Using bcrypt (RECOMMENDED)
import bcrypt

def hash_password(password):
    salt = bcrypt.gensalt(rounds=12)  # Work factor 12
    return bcrypt.hashpw(password.encode(), salt)

def verify_password(password, stored_hash):
    return bcrypt.checkpw(password.encode(), stored_hash.encode())


# Using Argon2 (BEST - NIST RECOMMENDED)
from argon2 import PasswordHasher

ph = PasswordHasher(
    time_cost=3,
    memory_cost=65536,
    parallelism=4,
    salt_len=16,
    hash_len=32
)

def hash_password(password):
    return ph.hash(password)

def verify_password(password, stored_hash):
    return ph.verify(stored_hash, password)
```
