# Password Cracking Methodology

## 1. Introduction to Password Cracking

Password cracking is the process of recovering passwords from data that has been stored or transmitted in a system. In this project, we demonstrate how vulnerable weak password hashing can be.

### Why We Crack Passwords (Legitimate Use Cases)
- **Penetration Testing** - Authorized security testing

## 2. Password Hashing Basics

### 2.1 What is Password Hashing?

```
Password: "password"
        ↓
    Hashing Algorithm (e.g., MD5)
        ↓
Hash: "5f4dcc3b5aa765d61d8327deb882cf99"
```

### 2.2 Properties of Good Hash Functions

| Property | Description |
|----------|-------------|
| One-way | Cannot reverse the hash to get password |
| Deterministic | Same input always produces same output |
| Fast computation | Hashing should be quick |
| Avalanche effect | Small change = completely different hash |
| Collision resistance | Hard to find two inputs with same hash |

### 2.3 Problems with MD5

```
⚠️ KNOWN VULNERABILITIES IN MD5:
• Collision attacks (2004) - Two different files same hash
• Pre-computed rainbow tables - Reverse lookup tables
• Fast computation - Good for passwords = BAD (easily brute-forced)
• No salt - Same password = same hash every time
• GPU acceleration - Can compute billions of MD5 per second
```

## 3. Attack Methods

### 3.1 Dictionary Attack

**Definition**: Uses a list of common passwords to attempt to crack hashes.

```
Wordlist:           Target Hashes:
┌──────────────┐   ┌────────────────────────────┐
│ password     │   │ 5f4dcc3b5aa765d61d8327...  │ → password ✓
│ 123456       │   │ e10adc3949ba59abbe56e0...  │ → 123456 ✓
│ admin        │   │ 827ccb0eea8a706c4c34a1...  │ → 12345 ✓
│ letmein      │   │ 202cb962ac59075b964b07...  │ → 123 ✓
│ (500M more)  │   │ ...                        │
└──────────────┘   └────────────────────────────┘
```

**Tools**: Hashcat, John the Ripper, CrackStation
**Success Rate**: 30-70% depending on wordlist quality

### 3.2 Rainbow Table Attack

**Definition**: Pre-computed lookup tables that reverse cryptographic hash functions.

```
Pre-computation Phase (One time):
┌────────────┐    MD5    ┌─────────────────┐
│ password   │ ────────→ │ 5f4dcc3b5...    │
│ 123456     │ ────────→ │ e10adc394...    │
│ admin      │ ────────→ │ d033e22ae...    │
│ ...        │           │                 │
└────────────┘           └─────────────────┘
         ↓ Rainbow Table Storage (GBs)
         ↓
Attack Phase (Fast):
Target: 5f4dcc3b5aa765d61d8327deb882cf99
        ↓
Lookup in Rainbow Table
        ↓
Result: password ✓
```

**Why it works**: Pre-computes all common passwords once, then instant lookup.

### 3.3 Brute Force Attack

**Definition**: Systematically tries every possible combination.

```
Charset: a-z (26 chars), 0-9 (10 chars), A-Z (26 chars) = 62 total

Attempt Calculation:
• 4 characters: 62^4 = 14,776,336 combinations
• 6 characters: 62^6 = 56,800,235,584 combinations
• 8 characters: 62^8 = 218,340,105,584,896 combinations

Time Estimate (GPU cluster):
• 4 chars: < 1 second
• 6 chars: ~30 minutes
• 8 chars: ~4 years

⚠️ Warning: Exponential growth makes this impractical for long passwords
```

### 3.4 Hybrid Attack

**Definition**: Combines dictionary attack with mutation rules.

```
Base Word     Mutations Applied      Result
─────────────────────────────────────────────
password      → number append        → password1
              → leet speak           → p@ssw0rd
              → year suffix          → password2023
              → capital first        → Password
              → duplicate            → passwordpassword
```

**Tools**: Hashcat rules, John the Ripper masks

## 4. Tools Used

### 4.1 Hashcat

**Description**: World's fastest and most advanced password recovery tool.

```bash
# Basic syntax
hashcat -m [hash_type] -a [attack_mode] [hash_file] [wordlist]

# MD5 dictionary attack
hashcat -m 0 -a 0 hashes.txt wordlist.txt

# MD5 brute force (8 chars, alphanumeric)
hashcat -m 0 -a 3 hashes.txt ?a?a?a?a?a?a?a?a

# MD5 with rules
hashcat -m 0 -a 0 hashes.txt wordlist.txt -r rules/best64.rule
```

| Parameter | Meaning |
|-----------|---------|
| -m 0 | MD5 hash type |
| -a 0 | Dictionary attack |
| -a 3 | Brute-force attack |
| ?a | All printable characters |

### 4.2 John the Ripper

**Description**: Multi-platform password cracking tool.

```bash
# Auto-detect hash type
john hashes.txt

# Specify format
john --format=raw-md5 hashes.txt

# Use wordlist
john --format=raw-md5 hashes.txt --wordlist=wordlist.txt

# Show cracked passwords
john --show hashes.txt
```

### 4.3 HashID

**Description**: Tool to identify hash types.

```bash
# Identify hash
hashid 5f4dcc3b5aa765d61d8327deb882cf99

# Output
Analyzing '5f4dcc3b5aa765d61d8327deb882cf99'
[+] MD5
[+] MD4
[+] MD2
[+] NTLM
```

## 7. Mitigation Against Cracking

### 7.1 Algorithm Selection

| Algorithm | Time to Crack (8 char password) | Recommendation |
|-----------|--------------------------------|-----------------|
| MD5 | < 1 second | ❌ Never use |
| SHA-1 | < 1 second | ❌ Deprecated |
| SHA-256 | < 1 second | ❌ Not for passwords |
| bcrypt | ~3 years | ✅ Good (cost=10) |
| Argon2id | ~10 years | ✅ Best |

### 7.2 Implementation Checklist

```python
# BAD - Vulnerable
import hashlib
hashlib.md5(password.encode()).hexdigest()

# GOOD - Secure (using bcrypt)
import bcrypt
hashed = bcrypt.hashpw(password.encode(), bcrypt.gensalt())

# BEST - Modern (using Argon2)
from argon2 import PasswordHasher
ph = PasswordHasher()
hash = ph.hash(password)
```

### 7.3 Salting

```python
# Without salt (BAD)
hash("password") → always "5f4dcc3b5aa765d61d8327deb882cf99"

# With unique salt (GOOD)
salt1 = random_bytes(16)
salt2 = random_bytes(16)
hash("password", salt1) → "a1b2c3d4..."
hash("password", salt2) → "e5f6g7h8..."
# Same password, different hashes!
```

## 8. Summary

### Key Takeaways

1. **MD5 is broken** - Never use for password storage
2. **Salt is essential** - Prevents rainbow table attacks
3. **Length > Complexity** - 12+ character passphrase is stronger than "P@ssw0rd1!"
4. **Modern algorithms** - Use bcrypt or Argon2
5. **Rate limiting** - Prevents brute force attacks

### What We Demonstrated

- ✅ Dictionary attack cracked 100% of passwords
- ✅ Rainbow table attack is possible without salt
- ✅ Brute force is instant for short passwords
- ✅ Weak passwords = instant compromise
