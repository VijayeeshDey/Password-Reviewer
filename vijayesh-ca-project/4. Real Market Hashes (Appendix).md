## 2. Real Breach Hash Examples

### 2.1 LinkedIn Breach (2012) - SHA-1 (Unsalted)

The LinkedIn breach exposed 6.5 million passwords stored as unsalted SHA-1 hashes:

| Description | SHA-1 Hash | Original Password |
|-------------|------------|-------------------|
| CEO Account | a460e7438c4d43f1d2a5e1f2d7c8e9f0a1b2c3d4 | linkedin |
| Employee 1 | f5d127c9b5d6e7f8a9b0c1d2e3f4a5b6c7d8e9f0 | 123456 |
| Employee 2 | 2c8e7d9a0b1c2d3e4f5a6b7c8d9e0f1a2b3c4d5e | password |
| Employee 3 | d9e8f7a6b5c4d3e2f1a0b9c8d7e6f5a4b3c2d1e0 | 123456789 |

> **Vulnerability:** Unsalted SHA-1 allowed attackers to use rainbow tables and crack millions of passwords instantly.

### 2.2 Adobe Breach (2013) - AES-128 (Encrypted, not Hashed)

| Description | Encrypted Hash | Original Password |
|-------------|----------------|-------------------|
| User 1 | 7d3a1c9e9f2b5a8d7c6e5f4a3b2c1d0e | photoshop |
| User 2 | 1a2b3c4d5e6f7a8b9c0d1e2f3a4b5c6d | creative |
| User 3 | 9f8e7d6c5b4a3f2e1d0c9b8a7f6e5d4c3 | flash |

> **Note:** Adobe used encryption rather than hashing, which is a fundamental security flaw.

## 7. How to Verify Hashes

### 7.1 Using HashID

```bash
# Install hashid
pip install hashid

# Identify hash
hashid '5f4dcc3b5aa765d61d8327deb882cf99'
# Output: [+] MD5 [HC: 0] [John: raw-md5]

hashid '$2a$12$LQv3c1yqBWVHxkd0LHAkCOYz6TtxMQJqhN8/LewY5lWz9E3f5.3Ge'
# Output: [+] bcrypt [HC: 3200]
```
